# 建造工作流

## 总体流程（6 阶段）

```
Stage 1: 调研 (2 Agent 并行)
  ├─ Agent A: 真实园林参考
  └─ Agent B: MC 最佳实践
         ↓
Stage 2: 综合出方案 (1 Agent)
  └─ 输出: config.py + 总平面图
         ↓
Stage 3: Review (1 Agent)
  └─ 碰撞检查 + 水面面积验证
         ↓
Stage 4: 批判 (1 Agent)
  └─ 文学/规制/空间/游线审查
         ↓
Stage 5: 定稿 (1 Agent 综合修订)
         ↓
Stage 6: 建造 (按 Phase 顺序, 每步 2 Agent)
  Phase 1: 水面+地形 (最先! 水面≥28%)
  Phase 2: 围墙+廊道骨架
  Phase 3: 建筑 (一体化函数, 不分开建)
  Phase 4: 细节 (植被/灯笼/做旧/家具)
```

## 单栋建筑建造流程

```
1. 从 config 读取参数 (cx, cz, size, roof_type, ground_y)
2. 清除整个区域 (fill air, 带1格余量)
3. 统一铺地面 (连续, 无断裂)
4. 建台基 (stone_bricks, 含散水带)
5. 立柱 (crimson_stem + polished_andesite 柱础)
6. 架梁 (dark_oak_planks)
7. 砌墙 (white_concrete, 留门洞≥3格宽)
8. 盖顶 (stone_brick_stairs, 按类型选攒尖/歇山/悬山)
9. 加飞檐 (dark_oak_stairs 倒置)
10. 放栏杆 (crimson_fence, 入口不放)
11. 挂灯笼 (lantern)
12. 输出接口点坐标 (供衔接用)
```

## 建筑群一体化建造（关键改进！）

当两栋建筑相邻时（如门厅+庭院），必须：
- 写成一个函数 build_xxx_complex()
- 先清除整个范围
- 统一铺地面
- 墙体共享（门厅东西墙延续到庭院矮墙）
- 屋顶只覆盖该有顶的部分

## 衔接六原则

1. **永不直通** — 至少1次转折
2. **转折有景** — 石组/花丛/灯笼
3. **三层过渡** — 台基→散水(1格andesite)→铺装
4. **植被边界** — 路两侧不接裸草
5. **高差用阶** — 每1格用 stone_brick_stairs
6. **水上规则** — 短距≤5格用汀步，长距用桥

## 验证检查清单

建造后必查：
- [ ] 水面面积 ≥ 28%
- [ ] 所有门洞 ≥ 3宽 × 3高
- [ ] 廊道不穿水面
- [ ] 建筑不在水里
- [ ] 屋顶完整（没被 fix 脚本清掉）
- [ ] 叶子有 persistent=true
- [ ] 楼梯 facing 方向正确
- [ ] 地面连续无断裂

## Agent 配置

| 任务类型 | Agent 数量 | 角色 |
|---------|-----------|------|
| 调研 | 2-4 | 各查不同方向 |
| 设计方案 | 1 | 综合调研出稿 |
| Review | 1 | 碰撞+通行性+代码 |
| 批判 | 1 | 审美+文学+规制 |
| 建造代码 | 1 | 写一体化函数 |
| 衔接设计 | 1 | 精确坐标+转折景观 |

每个修改至少 2 Agent（主推+Review），关键决策用 3 Agent（+批判）。
