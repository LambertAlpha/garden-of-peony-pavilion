# Prompt 模板库

## 1. 建筑主推 Agent Prompt 模板

```
你是 Minecraft 中式建筑师。写一个脚本建造 [建筑名称]。

## 必读文件
- config_v3.py — 坐标和参数
- builder.py — 建造 API
- blocks.py — 材质映射

## 建筑参数
- 位置: (cx, cz), ground_y
- 尺寸: size_x × size_z
- 屋顶: [攒尖/歇山/悬山/硬山]
- 朝向: [面水/面南/面东...]

## 一体化建造要求
1. 一个函数建完全部（含台基+柱+墙+屋顶+栏杆+地面）
2. 先清除整个区域再建
3. 地面连续无断裂
4. 入口 ≥ 3格宽 × 3格高
5. 所有叶子 persistent=true

## 效率要求
- 台基/屋顶/墙体用 fill
- 目标命令 < [N] 条

## 连接器声明
输出建筑的接口点坐标（北/南/东/西出入口的精确位置和Y高度）

只写文件不执行。
```

## 2. Review Agent Prompt 模板

```
Review [文件路径]。
Also Read config_v3.py 验证坐标。

## 碰撞检查（最重要！）
1. 建筑四角是否在水面内？代入椭圆公式验证
2. 是否与相邻建筑重叠？列出所有相邻建筑范围
3. 新结构是否破坏已有廊道/桥梁？

## 通行性检查
4. 所有门洞 ≥ 3格宽 × 3格高？
5. 楼梯 facing 方向是否正确？
6. 入口是否朝向来路？

## 代码检查
7. fill 坐标 x1<=x2, y1<=y2, z1<=z2？
8. 所有叶子 persistent=true？
9. 总命令数预估

用中文，简洁。输出: ✅/❌ + 修复建议
```

## 3. 批判 Agent Prompt 模板

```
你是严苛的中式园林评论家。批判 [建筑/方案名]。

## 批判维度
1. 比例：和周围建筑协调吗？
2. 文学：有原文依据吗？名字贴切吗？
3. 规制：屋顶等级是否僭越？
4. 空间：和水面的关系对吗？
5. 游线：在叙事动线中的位置合理吗？
6. 遗漏：缺了什么关键元素？

搜索 web 找参考。用中文，越犀利越好。
```

## 4. 衔接调研 Agent Prompt 模板

```
设计 [建筑A] 和 [建筑B] 之间的衔接。

## 数据
- A: 位置(cx,cz), 出口在哪面, Y高度
- B: 位置(cx,cz), 入口在哪面, Y高度

## 输出
- 路径类型（有顶廊/石径/桥/汀步）
- 宽度
- Y高差处理（台阶数量和facing）
- 转折点景观元素
- 精确坐标

## 原则
- 不直通，至少1次转折
- 转折处有景
- 三层过渡：台基→散水→铺装
- 路两侧有植被边界
```
